AMD SEV-SNP Remote Attestation: Cryptographic VM Identity Verification

What is AMD SEV-SNP Remote Attestation?

AMD SEV-SNP (Secure Encrypted Virtualization – Secure Nested Paging) remote attestation is a cryptographic process that verifies the identity and integrity of a virtual machine running in a genuine AMD SEV-SNP confidential computing environment. This skill provides a complete workflow for proving that your VM is authentic and hasn’t been tampered with.

When to Use SEV-SNP Attestation

You should use this skill when:

  • Proving to remote parties that your VM is running in a genuine AMD SEV-SNP environment
  • Verifying VM integrity before trusting it with sensitive secrets or cryptographic keys
  • Checking if SEV-SNP is properly configured and available on your system
  • Generating attestation reports for third-party verification
  • Validating the complete AMD certificate chain (ARK → ASK → VCEK)
  • Debugging attestation failures or certificate-related issues

The Complete Attestation Workflow

The SEV-SNP attestation process follows a structured 6-step workflow that ensures comprehensive verification:

Step 1: Detection – Is SEV-SNP Available?

The first step checks whether the system supports SEV-SNP and if the necessary device files are accessible. This requires root access or membership in the ‘sev’ group to access /dev/sev-guest.

Step 2: Generate Report

If SEV-SNP is available, the system generates a cryptographic attestation report. This report contains measurements of the VM’s current state, including firmware, hypervisor, and VM configuration.

Step 3: Display Report Information

The generated report is displayed with detailed information about the VM’s Trusted Computing Base (TCB) level, launch configuration, and other critical measurements.

Step 4: Fetch AMD Certificates

The system retrieves the AMD certificate chain from AMD’s Key Distribution Service (KDS) at https://kdsintf.amd.com. This includes the AMD Root Key (ARK), AMD Signing Key (ASK), and the VCEK certificate.

Step 5: Verify Certificate Chain

The certificate chain is validated to ensure it follows the proper hierarchy: ARK (self-signed) → ASK → VCEK. This confirms that the certificates are authentic and issued by AMD.

Step 6: Verify Report Signature

Finally, the attestation report’s signature is verified using the VCEK certificate. This confirms that the report was indeed generated by a genuine AMD SEV-SNP platform.

Quick Start Commands

Getting started with SEV-SNP attestation is straightforward:

Check if SEV-SNP is Available
./scripts/detect-sev-snp.sh

Run Full Attestation
./scripts/full-attestation.sh [output_dir]

The full attestation script runs all six steps and outputs either “PASSED” or “FAILED” based on the verification results.

Individual Step Scripts

Each step can be run independently for debugging or custom workflows:

scripts/detect-sev-snp.sh - Check SEV-SNP availability
scripts/generate-report.sh <output_dir> - Generate attestation report with nonce
scripts/fetch-certificates.sh <report_file> <output_dir> - Fetch AMD certificates from KDS
scripts/verify-chain.sh <certs_dir> - Verify certificate chain
scripts/verify-report.sh <report_file> <certs_dir> - Verify report signature

Prerequisites and Installation

Before running the attestation workflow, ensure you have the following:

  • snpguest: Rust CLI from virtee/snpguest (install with cargo install snpguest)
  • openssl: For certificate operations
  • curl: For fetching certificates from AMD KDS
  • Root access: Required to access /dev/sev-guest or membership in the ‘sev’ group

Technical Details

Understanding the technical aspects helps with troubleshooting and implementation:

  • AMD KDS URL: https://kdsintf.amd.com
  • Certificate Chain: ARK (self-signed) → ASK → VCEK
  • Report Signature: ECDSA P-384
  • Device: /dev/sev-guest (requires root or sev group membership)

Report Fields and Error Handling

The attestation report contains detailed fields about the VM’s configuration and measurements. Common error codes are documented to help troubleshoot issues during the attestation process.

Manual Verification Options

For advanced users, there’s an OpenSSL-based verification option that allows attestation without the snpguest CLI, providing more control over the verification process.

  • Authentication: Confirms VMs are running on genuine AMD hardware
  • Remote Trust: Enables third parties to verify VM authenticity
